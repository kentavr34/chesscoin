# BIBLE.md — Claudia Bot (Python async)

## ЗАПРЕЩЕНО
- Использовать `time.sleep()` в async коде — только `asyncio.sleep()`
- Блокирующие IO операции в корутинах без `run_in_executor`
- Отправлять несколько сообщений там где достаточно одного
- Глобальные переменные для состояний — только session store
- Перехватывать `Exception` без логирования (`except Exception: pass`)
- Хардкодить user_id, chat_id, токены — только через `.env`

## АРХИТЕКТУРА
- Состояния: IDLE → COLLECTING → CONFIRMING (3-state machine)
- Media groups: буфер `_mg_buffer` + asyncio таймер 1.5s
- Intent: `classify_intent()` через DeepSeek (max 5 токенов, temperature=0)
- `send_reply(update, text, voice_text=None)` — голос только если передан `voice_text`
- Session store: `get_session()` / `set_session()` — атомарные операции

## ГОЛОС
- `voice_text` — короткая разговорная фраза (до 15 слов)
- Не читать вслух весь текст задачи или PMI-структуру
- Примеры: "Поняла, принято. Скажи окей когда готово." / "Задача в работе."

## КЛАУДИЯ — ЖЕНСКИЙ РОД
- "Поняла", "приняла", "начинаю", "готова" — всегда женский род
- НЕ "Понял", НЕ "принял", НЕ "готов"

## УВЕДОМЛЕНИЯ
- Одно сообщение "начинаю задачу" — сразу с именем воркера
- Финальный отчёт: файлы + summary + BIBLE-аудит
- При ошибке — конкретное сообщение, не общее "что-то пошло не так"

## ПРОМПТЫ ДЛЯ АГЕНТОВ
- Всегда начинать с: "Перед изменениями — прочитай логи ошибок"
- Порядок: логи → исходники → минимальные изменения
- Запрещено менять файлы вслепую без анализа реальной ошибки

## ПРОЦЕСС ЗАДАЧИ
1. `pending` → `processing` (создаём ветку)
2. Одно `send_telegram` с именем воркера
3. Агент работает (Aider или Claude Code)
4. BIBLE-аудит диффа
5. `audit` → отчёт Кенану
6. `/deploy_N` или `/revert_N` — деплой или откат
