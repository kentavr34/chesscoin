## 1. КОНЦЕПТУАЛЬНЫЕ ИЗМЕНЕНИЯ

### Было → Стало

| Было | Стало |
|------|-------|
| «Нации» (Nations) — раздел с кланами | «Войны» (Wars) — раздел с войнами между странами |
| Кланы = произвольные группы с именем | Страны = реальные страны мира (все, ~195 шт.) |
| Вступление требует одобрения Commander | Вступление в страну — мгновенное, без одобрения |
| Commander назначается вручную / первый участник | Главнокомандующий = игрок с наибольшим количеством побед в стране |
| Максимум 100 участников | Максимум 1000 бойцов на страну |
| Кланы создаются пользователем | Страны предустановлены в БД (сидирование) |
| ClanWar — между кланами пользователей | War — между странами, объявляет только Главнокомандующий |
| Кланы в разделе «Войны» | Кланы → раздел «Турниры» (реферальная система, позже) |
| Нет истории войн | История прошедших войн с партиями и результатами |
| Нет просмотра партий | Просмотр и сохранение партий из истории войн |

### Что такое «Войны» — ключевая механика

- **Страны** — предустановленные, реальные страны мира.
- **Боец** — любой игрок, вступивший в страну. Можно сменить страну («трансфер»).
- **Главнокомандующий** — боец с наибольшим числом побед в составе данной страны. Если один участник — он автоматически Главнокомандующий.
- **Объявление войны** — только Главнокомандующий может вызвать другую страну. После объявления война стартует сразу (без «принятия»).
- **Во время войны** — любой боец своей страны может вызвать игрока страны-противника на баттл без ставки (через раздел «Войны»). Победа засчитывается в казну страны.
- **Вне войны** — обычные баттлы в разделе «Баттлы» идут в личный счёт, казна страны не пополняется.
- **Наблюдатели** — любой игрок (даже без страны) видит активные войны и историю.
- **Рейтинг стран** — по количеству побед в казне (по умолчанию алфавитный, растёт при победах).

## ЭТАП 2 — БАЗА ДАННЫХ

**Файл:** `backend/prisma/schema.prisma`

### 2.1. Таблица стран (Country)

Создать новую модель **Country** (или переименовать/переиспользовать Clan):

```prisma
model Country {
  id          String @id @default(uuid())
  code        String @unique   // "RU", "BR", "DE", "US"...
  nameRu      String           // Россия
  nameEn      String           // Russia
  flag        String           // 🇷🇺 (emoji флага)

  treasury    BigInt @default(0)   // Казна — суммарные победы
  wins        Int @default(0)      // Всего побед
  losses      Int @default(0)      // Всего поражений

  maxMembers  Int @default(1000)

  members     CountryMember[]
  warsAsAttacker CountryWar[] @relation("AttackerCountry")
  warsAsDefender CountryWar[] @relation("DefenderCountry")

  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt
}
```

### 2.2. Бойцы страны (CountryMember)

```prisma
model CountryMember {
  id          String @id @default(uuid())

  countryId   String
  country     Country @relation(fields: [countryId], references: [id])

  userId      String @unique    // Один игрок — одна страна
  user        User @relation(fields: [userId], references: [id])

  warWins     Int @default(0)   // Победы в войнах (определяет Главнокомандующего)
  warLosses   Int @default(0)

  joinedAt    DateTime @default(now())

  @@index([countryId, warWins])
}
```

**Главнокомандующий** — НЕ хранится отдельным полем, определяется динамически:
```sql
SELECT userId FROM CountryMember
WHERE countryId = :id
ORDER BY warWins DESC, joinedAt ASC
LIMIT 1
```

### 2.3. Войны между странами (CountryWar)

```prisma
model CountryWar {
  id               String @id @default(uuid())

  attackerCountryId String
  attackerCountry   Country @relation("AttackerCountry", fields: [attackerCountryId], references: [id])
  defenderCountryId String
  defenderCountry   Country @relation("DefenderCountry", fields: [defenderCountryId], references: [id])

  status           CountryWarStatus @default(IN_PROGRESS)
  attackerWins     Int @default(0)
  defenderWins     Int @default(0)

  prizePerWin      BigInt @default(100)  // Монет за победу в казну

  duration         Int              // Секунды (1ч, 12ч, 24ч, 7д, 30д)
  startedAt        DateTime @default(now())
  endAt            DateTime         // startedAt + duration
  finishedAt       DateTime?
  winnerCountryId  String?

  declaredByUserId String           // userId Главнокомандующего

  battles          WarBattle[]      // Индивидуальные партии в рамках войны

  createdAt        DateTime @default(now())
}

enum CountryWarStatus {
  IN_PROGRESS
  FINISHED
}
```

### 2.4. Партии в рамках войны (WarBattle)

```prisma
model WarBattle {
  id           String @id @default(uuid())

  warId        String
  war          CountryWar @relation(fields: [warId], references: [id])

  sessionId    String @unique    // Ссылка на обычную Session (chess game)
  session      Session @relation(fields: [sessionId], references: [id])

  attackerId   String            // userId атакующей стороны
  defenderId   String            // userId защищающейся стороны
  attackerCountryId String
  defenderCountryId String

  winnerId     String?
  winnerCountryId String?

  status       String @default("IN_PROGRESS")  // IN_PROGRESS | FINISHED

  createdAt    DateTime @default(now())
  finishedAt   DateTime?
}
```

### 2.5. Сохранённые партии пользователя (SavedGame)

```prisma
model SavedGame {
  id        String @id @default(uuid())
  userId    String
  user      User @relation(fields: [userId], references: [id])
  sessionId String
  session   Session @relation(fields: [sessionId], references: [id])
  savedAt   DateTime @default(now())

  @@unique([userId, sessionId])
}
```

## ЭТАП 3 — BACKEND API

**Файлы:** `backend/src/routes/nations.ts` → переименовать в `wars.ts`, `backend/src/index.ts`

### 3.1. Новые эндпоинты

| Метод | Путь | Auth | Описание |
|-------|------|------|----------|
| `GET` | `/wars/countries` | JWT | Список всех стран (сортировка: wins DESC, затем алфавит) |
| `GET` | `/wars/countries/:id` | JWT | Данные страны + список бойцов |
| `POST` | `/wars/countries/:id/join` | JWT | Вступить в страну (мгновенно, без одобрения) |
| `POST` | `/wars/countries/leave` | JWT | Покинуть страну |
| `GET` | `/wars/my-country` | JWT | Моя страна + мой статус бойца |
| `GET` | `/wars/active` | JWT | Активные войны (видят все) |
| `GET` | `/wars/history` | JWT | История завершённых войн |
| `GET` | `/wars/:warId` | JWT | Детали войны + список партий |
| `POST` | `/wars/declare` | JWT | Главнокомандующий объявляет войну |
| `POST` | `/wars/:warId/challenge` | JWT | Боец вызывает противника в рамках войны |
| `POST` | `/wars/intro-seen` | JWT | Отметить что показали вводное окно |
| `POST` | `/wars/games/:sessionId/save` | JWT | Сохранить партию в профиль |

### 3.2. Логика эндпоинтов

#### GET `/wars/countries`
```
- Все страны из БД
- Сортировка: wins DESC, nameRu ASC
- Каждая страна: code, nameRu, nameEn, flag, treasury, wins, losses, memberCount (реальное), commanderId (userId первого в ORDER BY warWins DESC)
- Включить: myMembership (если пользователь — боец этой страны)
```

#### GET `/wars/countries/:id`
```
- Данные страны
- Список бойцов: ORDER BY warWins DESC, joinedAt ASC
- Первый боец = Главнокомандующий (isCommander: true)
- Для каждого бойца: userId, firstName, lastName, elo, league, warWins, warLosses, joinedAt
- Активная война (если есть)
```

#### POST `/wars/declare`
```
Body: { defenderCountryId, duration }
- Проверить: пользователь — Главнокомандующий своей страны
- Проверить: нет активной войны у обеих стран
- Создать CountryWar: status=IN_PROGRESS, endAt=now()+duration
- Отправить уведомление Главнокомандующему страны-противника
- Запустить таймер завершения войны (Redis TTL)
- Ответ: { war }
```

#### POST `/wars/:warId/challenge`
```
Body: { opponentUserId }
- Проверить: война IN_PROGRESS
- Проверить: пользователь — боец страны-участницы войны
- Проверить: opponent — боец страны-противника
- Создать обычную Session (type=BATTLE, без ставки)
- Создать WarBattle
- Отправить Socket.io уведомление opponent
- Ответ: { session }
```

#### Завершение войны (cron/timer)
```
При endAt: обработчик финализирует войну
- Считает attackerWins vs defenderWins
- Определяет winnerCountryId
- Пополняет казну победителя (prizePerWin × wins)
- Отправляет уведомление всем бойцам обеих стран
- Обновляет рейтинг (wins/losses) обеих стран
```

### 3.3. Логика Главнокомандующего

```typescript
// Вспомогательная функция
async function getCommander(countryId: string): Promise<string | null> {
  const top = await prisma.countryMember.findFirst({
    where: { countryId },
    orderBy: [{ warWins: 'desc' }, { joinedAt: 'asc' }],
    select: { userId: true }
  });
  return top?.userId ?? null;
}
```

## ЭТАП 5 — UI-ИСПРАВЛЕНИЯ И ПОЛИРОВКА

### 5.1. Все модальные окна — крестик обязателен

**Затронутые файлы:** все `*Modal*` компоненты в `frontend/src/components/`

- Каждый модал/popup/нижний лист → добавить кнопку [✕] в правом верхнем углу
- Крестик не должен исчезать сам по себе через N секунд
- Исключение: технические уведомления (Toast) с коротким текстом — 3-5 сек, без крестика

### 5.2. Кликабельность профилей

- Аватар/имя любого игрока → ссылка на `/profile/:userId`

- Во всех местах где показывается «бойцов в стране»: брать реальный `_count.members` из БД

### 5.4. Главнокомандующий

- В списке бойцов: первым идёт Главнокомандующий

## ЭТАП 6 — ИНТЕГРАЦИЯ TON-БЛОКЧЕЙНА

> ⚠️ Это отдельный крупный этап, требующий настройки смарт-контрактов и отдельного юридического/технического ревью. Ниже — план архитектуры.

### 6.1. Выпуск токена CHESSCOIN на TON

- Эмиссия: **100 миллиардов** CHESSCOIN
- Тип: Jetton (стандартный fungible token на TON)
- Инструмент выпуска: TON Minter (https://minter.ton.org) или через `@ton-community/jetton` смарт-контракт
- Токены размещаются на **отдельном кошельке-казначействе** (treasury wallet)
- Из казначейства идёт начисление при выводе

### 6.2. Подключение кошелька

**Кнопка «Подключить кошелёк»** — в разделе «Профиль» или при первой попытке вывода:

```
┌──────────────────────────────────────────┐
│  💼 Подключить кошелёк             [✕]  │
├──────────────────────────────────────────┤
│  Для вывода монет на внешний кошелёк    │
│  необходимо его подключить.             │
│                                         │
│  Стоимость подключения: 1 TON           │
│  (единоразово для каждого кошелька)     │
│                                         │
│  [📱 Telegram Wallet]                   │
│  [🔗 Tonkeeper]                         │
│  [🔗 MyTonWallet]                       │
└──────────────────────────────────────────┘
```

**Логика:**
- Используем **TON Connect 2.0** (библиотека `@tonconnect/ui-react`)
- При подключении: проверить баланс TON → списать 1 TON на наш кошелёк
- Если < 1 TON: уведомление «Недостаточно TON для подключения»
- Сохранить `walletAddress` в `User.walletAddress` (добавить поле в схему)
- Переподключение другого кошелька: сначала «Отключить», затем повторное подключение = снова 1 TON

### 6.3. Покупка монет (TON/USDT → CHESSCOIN)

```
Пользователь           Платформа              Backend
    │                      │                     │
    │ Нажал «Купить»        │                     │
    │──────────────────────►│                     │
    │                      │ Создать invoice      │
    │                      │────────────────────► │
    │ TON-транзакция        │                     │
    │──────────────────────►│ (на наш TON-кошелёк)│
    │                      │                     │
    │                      │ Подтвердить tx       │
    │                      │────────────────────► │
    │                      │                     │ updateBalance(+CHESSCOIN)
    │ Монеты на балансе     │◄────────────────────│
    │◄──────────────────────│                     │
```

- Курс: определяется через `PlatformConfig.tokenPrice`
- CHESSCOIN зачисляются на **внутренний баланс** (в БД), не на кошелёк
- На кошелёк выводятся только по запросу

### 6.4. Вывод монет (CHESSCOIN → TON-кошелёк)

```
┌──────────────────────────────────────────┐
│  💸 Вывод монет                    [✕]  │
├──────────────────────────────────────────┤
│  Доступно: 15,000 ᚙ                    │
│  Кошелёк: UQA...xyz                    │
│                                         │
│  Сумма вывода: [__________] ᚙ          │
│                                         │
│  Комиссия сети TON: ~0.05 TON           │
│  Комиссия платформы: 0.5%               │
│  Вы получите: ≈ 14,925 ᚙ               │
│                                         │
│  [Вывести]                              │
└──────────────────────────────────────────┘
```

**Логика вывода:**
1. Проверить `walletAddress` (кошелёк подключён)
2. Списать `amount` с внутреннего баланса + 0.5% платформенной комиссии
3. Отправить Jetton-транзакцию с treasury-кошелька на `walletAddress`
4. Записать Transaction в БД

### 6.5. P2P торговля между игроками

**Механика:**
- Продавец выставляет лот: «X CHESSCOIN за Y TON»
- Покупатель отправляет TON → на кошелёк продавца
- Платформа берёт 0.5% комиссии в TON с каждой сделки
- CHESSCOIN списываются с баланса продавца → зачисляются на баланс покупателя (в БД)
- Реализуется через смарт-контракт-эскроу или через платформенный кошелёк-посредник

> Детальный план P2P-торговли — отдельный документ (Фаза 3)

### 6.6. Технические зависимости для TON

```bash
# Frontend
npm install @tonconnect/ui-react

# Backend
npm install @ton/ton @ton/core @ton/crypto
```

- Добавить `TREASURY_WALLET_MNEMONIC` в `.env` (секретная фраза кошелька-казначейства)
- Добавить `TONCONNECT_MANIFEST_URL` — URL `tonconnect-manifest.json`
- Создать `public/tonconnect-manifest.json` на домене приложения
