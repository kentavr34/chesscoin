## 🎮 ДЕТАЛЬНОЕ ОПИСАНИЕ РЕЖИМОВ

---

### 1. ДЖАРВИС (Игра с ботом)

**Путь:**
```
HomePage → кнопка "Джарвис" → GameSetupModal (уровень 1-20)
→ validateCanStartSession (макс 1 BOT-сессия)
→ useAttempt (тратится попытка)
→ createBotSession → Session IN_PROGRESS
→ GamePage: Stockfish Worker Pool (уровень → UCI_Elo)
→ game:move ↔ Stockfish ответ
→ game:over → processBotPayouts → BOT_WIN
→ jarvisLevel++ (если победа на текущем уровне)
→ checkGameTasks → WIN_BOT_N, FIRST_GAME, PLAY_N
→ applyReferralIncome (fire-and-forget)
```

**Особенности:**
- Уровни 1-20, каждый следующий открывается только после победы
- Stockfish: UCI_Elo от 1320 (Beginner) до 3190 (Mystic)
- Movetime: 50ms (уровень 1) → 30 000ms (уровень 20)
- Worker Pool: макс 20 одновременных процессов
- При проигрыше в Фазе 1: монеты не списываются
- Награды растут от 1 000 до 75 000 ᚙ за уровень

**Статус: ✅ ПОЛНОСТЬЮ РАБОТАЕТ**

---

### 2. БАТЛ P2P (Игра на ставку)

**Путь:**
```
BattlesPage → CreateBattleModal (ставка, цвет, время)
→ validateCanStartSession (макс 2 BATTLE)
→ useAttempt + updateBalance(-bet, BATTLE_BET)
→ createBattleSession → WAITING_FOR_OPPONENT
→ battles:list broadcast всем в лобби
→ Оппонент: joinBattleSession → updateBalance(-bet)
→ Оба: game:started, setTimer
→ game:move (по очереди, с таймером)
→ Победа матом / сдача / время вышло / ничья
→ processBattlePayouts:
   - победитель: winnerPayout = (bet×2) - 10% комиссии (BATTLE_WIN)
   - донат зрителей → победителю (если есть)
   - комиссия → platformReserve
   - ELO обоих обновляется
→ checkGameTasks → WIN_N, FIRST_GAME, PLAY_N
```

**Особенности:**
- Минимальная ставка: 1 000 ᚙ
- Комиссия стола: 10% от общего банка
- Скины создателя батла применяются обоим (S3)
- Зрители могут смотреть и донатить

**Статус: ✅ ПОЛНОСТЬЮ РАБОТАЕТ**

---

### 3. ДРУЖЕСКАЯ ИГРА (Friendly)

**Путь:**
```
Приглашение по ссылке → createFriendlySession
→ Оппонент: joinFriendlySession
→ Стандартная шахматная партия
→ Завершение: нет финансовых операций
→ checkGameTasks → FIRST_GAME, PLAY_N (без WIN_N)
```

**Особенности:**
- Нет ставок, нет наград за победу
- Засчитывается в задании PLAY_N и FIRST_GAME
- Проигрыш не сбрасывает WIN_STREAK (по дизайну)

**Статус: ✅ РАБОТАЕТ**

### 10. ТУРНИРЫ (5 типов)

**Типы:**
| Тип | Взнос | Приз (топ-1) | Требование |
|-----|-------|-------------|-----------|
| WEEKLY | 1 000 ᚙ | 10% банка | Нет |
| MONTHLY | 3 000 ᚙ | 20% банка | Нет |
| SEASONAL | 10 000 ᚙ | 30% банка | Нет |
| COUNTRY | 25 000 ᚙ | 60/30/10% | Членство в стране |
| WORLD | 50 000 ᚙ | 70/20/10% | Нет |

**Путь:**
```
TournamentsPage → выбрать → "Вступить"
→ $transaction: списание взноса + TournamentPlayer.create (race safe)
→ runTournamentMatchmaking → приватный батл + TournamentMatch
→ socket: tournament:match уведомление
→ Бот-сообщение если офлайн
→ Играем батл (isPrivate=true, не виден в лобби)
→ cron: checkTournamentResults (каждые 10 мин)
→ Призы начисляются топ-игрокам
→ Бот: TOURNAMENT_WIN уведомление
→ Авто-поражение через 24ч если не сыграл
```

**⚠️ ЗАМЕЧАНИЕ по WORLD:** В crons.ts есть условие `isYearly = t.type === 'COUNTRY'`, но для WORLD специальной ветки нет — WORLD попадает в else и получает **только 10%** (как WEEKLY). Это баг в распределении призов WORLD-турнира.

**Статус: ✅ РАБОТАЕТ (⚠️ WORLD призы 10% вместо 70%)**

### 14. РЕФЕРАЛЬНАЯ СИСТЕМА

```
Игрок А делится реф-ссылкой
→ Игрок Б регистрируется
→ Игрок Б играет первую партию → activateReferral()
  → Игроку А начисляется activationBonus (зависит от ранга)
  → RECRUIT: 0 ᚙ | PRIVATE (10 рефералов): 3 000 ᚙ | ... | EMPEROR: 40 000 ᚙ
→ Каждый раз когда Игрок Б выигрывает:
  → applyReferralIncome → Игроку А % от выигрыша
  → PRIVATE: 1% | ... | EMPEROR: 15%
→ Sub-referral (друг друга): 10% от победы
```

**Военный ранг растёт с числом рефералов:**
RECRUIT (0) → PRIVATE (10) → CORPORAL (50) → ... → EMPEROR (1 000 000)

**Статус: ✅ ПОЛНОСТЬЮ РАБОТАЕТ**

### 15. P2P БИРЖА

```
SELL: Создать ордер → монеты заморожены → покупатель платит TON
BUY: Создать ордер → продавец принимает → платит покупатель

Защиты:
- Race condition: updateMany WHERE status=OPEN
- Idempotency: @@unique(txHash)
- TON верификация: TonCenter API
- Лимит: 5 SELL + 5 BUY на пользователя
- Автоотмена: 30 дней → EXCHANGE_UNFREEZE
- Частичное исполнение: split ордер
```

**Статус: ✅ ПОЛНОСТЬЮ РАБОТАЕТ**

## 🔴 НАЙДЕННЫЕ БАГИ

### БАГ #5 — ВАЖНЫЙ: WORLD турнир получает 10% вместо 70%

**Файл:** `crons.ts` функция `checkTournamentResults`

**Проблема:**
```typescript
const isYearly = t.type === 'COUNTRY'; // только COUNTRY = 60/30/10%
// WORLD попадает в else → 10% (как WEEKLY!)
```

**Решение:**
```typescript
// Добавить ветку для WORLD
const prizes: [bigint, number][] =
  t.type === 'WORLD'
    ? [[totalPool * 70n / 100n, 0], [totalPool * 20n / 100n, 1], [totalPool * 10n / 100n, 2]]
  : isYearly  // COUNTRY
    ? [[totalPool * 60n / 100n, 0], [totalPool * 30n / 100n, 1], [totalPool * 10n / 100n, 2]]
  : t.type === 'SEASONAL'
    ? [[totalPool * 30n / 100n, 0]]
  : t.type === 'MONTHLY'
    ? [[totalPool * 20n / 100n, 0]]
    : [[totalPool * 10n / 100n, 0]]; // WEEKLY
```
