# Дизайн-система Аудит (2026-03-31)

## КРИТИЧНЫЕ (красный свет)

### 1. Отсутствует система доступности (WCAG/A11Y)
- Нет ARIA-меток в интерактивных элементах
- Нет семантического HTML для кнопок, модалей
- Отсутствуют tabindex и фокус-управление
- Влияние: неиспользуемо для скринридеров

### 2. Z-index система полностью нарушена
- Переменные определены (--z-modal: 300, --z-toast: 400)
- **НО:** компоненты их игнорируют (zIndex: 1000, 500, 9999)
- Конфликты: Toast (9999) > всего, Modal (300) < ConfirmModal (1000)
- Влияние: неработающие модалки, оверлеи друг на друге

## ВЫСОКИЙ ПРИОРИТЕТ

### 3. Дублирование модальных окон (12 компонент)
- Каждая переопределяет: position, zIndex, padding, borderRadius
- Нет базовой компоненты для переиспользования
- ConfirmModal, GameSetupModal, BadgeDetailModal, JarvisModal и т.д. - копипаст

### 4. Hardcoding цветов вместо CSS variables
- 50+ hex-цветов прямо в компонентах (#F5C842, #FF4D6A и т.д.)
- styles.ts имеет COLOR переменные, но не везде используются
- Несогласованность: один цвет может быть 5 разными способами

### 5. Отсутствует мобильная адаптация
- Нет @media запросов
- Нет мобильных breakpoints (mobile: 375px, tablet: 768px)
- Fixed высоты/ширины везде (maxWidth: 400px только)
- Telegram Mini App требует адаптации

### 6. Нет light/dark mode UI
- theme.ts имеет 7 полных тем (default, binance, neon_cyber и т.д.)
- Но нет переключения в UI
- Тема хардкодирована в компонентах

### 7. Боксовые тени не стандартизированы
- boxShadow дублируется везде без CSS переменных
- Нет --shadow-sm, --shadow-md, --shadow-lg

### 8. Spacing/padding/margin система не полностью используется
- SPACING константа есть (4, 8, 12, 16, 20, 24), но используется в 30% случаев
- Hardcoded значения: 3px, 5px, 6px, 7px, 11px, 13px, 18px, 22px, 28px, 32px
- Пример: ConfirmModal (padding: 28px 24px 22px - нестандарт)
