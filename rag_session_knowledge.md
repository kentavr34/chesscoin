## Система Клаудиа

### Что это
Автономный AI-менеджер проектов. Работает в Telegram. Принимает задачи от Кенана, анализирует через "Совет директоров" (LLM), передаёт Аидеру на выполнение.

### Архитектура
- Bot: /root/claudia/bot/main.py (python-telegram-bot, polling режим)
- Совет директоров: /root/claudia/crew_board.py (один LLM вызов, ~3 сек)
- Аидер runner: /root/claudia/aider_runner.py (выполнение кода)
- База: PostgreSQL, база claudia, таблицы: user_sessions, task_buffer, claudia_tasks
- Логи: /root/claudia/logs/bot.log

### LLM цепочка (failover)
1. Claude через OpenRouter (sk-or-v1-...)
2. Anthropic direct (sk-ant-api03-...)
3. DeepSeek V3 direct (sk-289f...)

### ASR (голос → текст)
Groq Whisper (gsk_X2M...) — бесплатно, ~1 сек

### TTS (текст → голос)
ElevenLabs eleven_multilingual_v2 (sk_7596...)

### Workflow задач
1. Кенан: /chesscoin → материалы → /ok
2. Совет директоров анализирует → PMI описание с этапами
3. Кенан: /ok → задача в очередь
4. Аидер создаёт ветку task-N, выполняет
5. Клаудиа присылает .diff файл + BIBLE аудит
6. Кенан: /deploy_N → merge в main → деплой

### Команды бота
- /chesscoin — задача для ChessCoin
- /task — задача для любого проекта
- /ok — подтвердить
- /cancel — отменить
- /status — статус задач
- /rag [вопрос] — запрос в LightRAG
- /deploy_N — задеплоить задачу N
- /revert_N — отменить задачу N

## Важные технические решения

### Почему polling, не webhook
Telegram кешировал старый IP (37.77.106.28) при смене на eVPS. Polling надёжнее.

### Почему DeepSeek вместо Claude для индексации
В 3 раза дешевле. Claude через OpenRouter для важных ответов Клаудии.

### Почему aider на ветке, не main
2-этапный процесс: выполнение → аудит → одобрение → деплой. Защита от плохого кода.

### BIBLE.md
Файл правил проекта на сервере (/opt/chesscoin/BIBLE.md). Клаудиа проверяет каждый diff против BIBLE перед отправкой Кенану.
