2. **Не упрощай компоненты** без причины (HomePage была уничтожена — помни).

6. **Доказательство = e2e на уровне пользователя**, не unit-тест.

8. **Redis всегда с номером БД.** Проект использует `DB=1`. `redis-cli` по
   умолчанию смотрит в `DB=0`. «Redis пуст» без `-n 1` — иллюзия.

## РАБОЧИЕ АЛГОРИТМЫ (опыт из предыдущих сессий)

### Алгоритм: «что-то упало на проде»

1. Сначала `docker ps` — какие контейнеры в `Restarting`, какие `Up`.
2. `docker logs <name> --tail 40` на упавший.
3. Ищи **корневую причину в логах**, не симптом. «Can't reach X» — значит
   X лежит, иди к X, не к тому, кто жалуется.
4. Если контейнер не стартует с `exit 127` и монтирует файл — проверь,
   не создал ли Docker **директорию** вместо файла на хосте. Классика.
5. Если Prisma ругается «operator does not exist: text = EnumName» — колонка
   в БД `text`, а схема ждёт `enum`. Нужна миграция `ALTER ... TYPE ... USING`.
6. Фикс на сервере **руками** = hotfix. **Обязательно** оформи миграцию/
   compose-правку в репо и запушь — иначе следующий reset всё откатит.

### Алгоритм: «SSH не пускает»

1. `test -r <ключ>` — ключ читается?
2. `ssh -o BatchMode=yes -o ConnectTimeout=10 ... "echo OK"` — что говорит:
   - `OK` → всё работает.
   - `Connection refused` → sshd лежит, reboot через панель.
   - `Connection timed out` → сеть.
   - `Permission denied (publickey)` → ключ не принят, пинг Кенану.
3. **Не пытайся** одно и то же 10 раз. Смени гипотезу.
4. В Claude Code sandbox мог блокировать команду — попроси allow-листнуть
   в `.claude/settings.local.json`.

### Алгоритм: «хочется поэкспериментировать с репо»

Не правь рабочую копию напрямую. Клонируй через `gh repo clone` в `/tmp/...`,
делай там ветку, копируй туда файлы из рабочей, коммить, пуш. Если сломала —
`rm -rf` копию, не рабочую. Рабочая копия = золото.

### Алгоритм: «как сделать reset-safe»

Каждый раз, когда руками чинишь что-то на сервере, спроси себя:
**«переживёт ли это hard reboot?»**. Если нет — оформи в docker-compose
healthcheck + `depends_on: condition: service_healthy`, или в post-boot
systemd-unit, или в entrypoint скрипт. Временный фикс = фикс, который
упадёт в следующий раз.

## ПАМЯТЬ (LightRAG)

Контекст, история сессий, задачи, дизайн-стандарты, уроки — в **LightRAG**:
- `query_graph("вопрос")` / `memory_search` — запросить.
- `save_to_memory("текст")` / `memory_remember` — сохранить.
- Веб: https://chesscoin.app/webui/

**Приоритет поиска:**
1. Сначала — `memory_search` / `query_graph`.
2. Не нашла → локальные файлы (README, `.claude/management/`, CLAUDE.md).
3. В последнюю очередь — сервер (SSH).

**RAG-цикл:** нашла что-то новое (в файлах или на сервере) — **обязательно**
`save_to_memory()`, чтобы в следующей сессии не искать заново.
