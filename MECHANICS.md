## 1. Сущности и связи

```
User (telegramId, balance ᚙ, attempts 0-3, jarvisLevel 1-20, militaryRank,
      referralCount, referralCreatorId, country?, activeTheme)
  │
  ├─ Sessions ─── Side (player, isWhite, status, timeLeft)
  │   ├─ BATTLE (1v1 stake, public|private)
  │   ├─ BOT (vs Jarvis)
  │   └─ FRIENDLY (без ставки)
  │
  ├─ TournamentPlayer ─── Tournament (WEEKLY|MONTHLY|SEASONAL|YEARLY|WORLD|COUNTRY)
  │                          └─ TournamentMatch (round, points, deadline)
  │
  ├─ CountryMember (clan/army) ─── Country (treasury, members, war stats)
  │                                  └─ CountryWar (attacker vs defender)
  │                                       └─ WarBattle (1v1 в рамках войны)
  │
  ├─ Transactions (BATTLE_BET, BATTLE_WIN, TOURNAMENT_WIN, COUNTRY_WAR_WIN, …)
  ├─ Referrals (level 1, level 2)
  ├─ SavedGames, Achievements, JarvisBadges, TournamentBadges
  └─ ShopPurchases (avatars, themes, board skins)
```

## 2. Режим JARVIS (vs ИИ)

### Поток партии
1. **HomePage** → клик «Jarvis» → `JarvisPlayModal`.
2. Один объединённый модал: уровень → выбор цвета (`Random` по умолчанию) →
   2 ряда таймов **`1 / 3 / 5 / 15 / 30 / 60`** мин (без 10) → кнопка «Играть».
3. **3-секундный обратный отсчёт** → переход на доску.
4. Игра против Stockfish соответствующего уровня (1–20).
5. **Финал**: 3-секундная задержка перед закрытием результата с уведомлением.
6. **Кнопка Реванш** в результате (только для BOT) — создаёт новую сессию с теми
   же параметрами.

### Экономика
- **Списание попытки** при старте партии (одна из 3, восстанавливается +1 каждые 8ч).
- При попытке играть без попыток → toast 3 сек на главной + модал покупки
  (1 000 ᚙ за попытку, макс. до 3 в день).
- **Награда за победу:** из `config.economy.botRewards[level]` (1k..1M).
- **За съеденные фигуры:** `BOT_PIECE` начисление в реальном времени (Фаза 1
  эмиссии).
- **Проигрыш боту:** теряется только попытка. Никаких `BOT_LOSS` штрафов
  (требует удаления в коде).
- **Доступ к уровню:** не выше `user.jarvisLevel + 1`.
- **Уведомление главным**: «нельзя играть с Jarvis, есть активная партия» —
  toast 3 сек, не «мигающая полоса».

### Бейджи
20 уровней: Beginner..Mystic. При первой победе над уровнем — бейдж в профиль.

## 3. Режим Батлы (1v1 на ставку)

### Создание
- Кнопка «Создать батл» в `BattlesPage`.
- `CreateBattleModal`: ставка (от 10 000 ᚙ), цвет (`Random|White|Black`), время
  (1/3/5/15/30/60 мин), переключатель **публичный/приватный**.
- Если нет попыток → кнопка КРАСНАЯ + модал покупки попыток.
- Если не хватает монет → модал «нет монет» → переход в магазин.
- После создания **игрок свободен**, батл висит в «Ожидают», push при подключении
  соперника.

### Публичный батл
- Виден в общем лобби сортировка: **Live сверху**, далее по убыв. ставки.
- Любой игрок может «Принять» (нужна попытка + ставка).
- Зрители смотрят live + могут донатить в кассу.
- **Касса = ставки обоих + донаты зрителей.**
- Победитель получает: `ставки×2 − 10% комиссия + донаты` (100% доната без
  комиссии).
- Каждый час бот публикует **топ батл с максимальной ставкой** в публичный канал.
  Если за час та же запись — пропустить. (Источник: `SPEC_2026-05-12 §2`.)

### Приватный батл
- Скрыт из общего лобби.
- Создатель отправляет другу **реферальную ссылку батла**
  `t.me/chessgamecoin_bot?start=battle_<sessionId>_<userId>`.
- Если получатель **впервые** на платформе — автоматически попадает в команду
  создателя батла. `Session.referralCreatorId` в схеме.
- Существующему игроку — всплывающее уведомление с кнопками `Принять/Отклонить`.
- **Настройка** в профиле: «Принимать вызовы от других игроков» (вкл/выкл).
- Касса = только взносы игроков (без зрительских донатов).
- 10% комиссия со стола, остальное — победителю.

### Карточка батла (3 этапа одного шаблона)
- **Этап 1 — Вызов** (никто не принял): аватар создателя + имя + ELO + ставка + время.
- **Этап 2 — Принят**: индикация противника.
- **Этап 3 — Live**: красный/жёлтый индикатор статуса, таймер.

Бейджи источника:
- **🏆 Турнир** (SVG-кубок, без эмодзи)
- **⚔️ Война** (SVG-мечи, без эмодзи)
- Обычный — без бейджа

### История партий — `BattleHistoryCard`
Единый шаблон для всех завершённых партий:
- [аватар] [бейдж результата ПОБЕДА/НИЧЬЯ/ПОРАЖЕНИЕ + дата + ставка] [аватар]
- Корона над аватаром победителя (SVG)
- Цвет рамки по результату (зелёная/синяя/красная)
- Клик в центр → PGN-просмотр

## 4. Режим Войны стран (кланы)

> Терминология: «клан» = «команда страны». Один игрок = одна страна.

### Состав
- 30 стран в seed.
- **Макс 100 бойцов** в одной стране (сейчас в коде 1000 — менять в Фазе 3).
- Вступление: взнос **10 000 ᚙ**, идёт в казну страны. **При выходе НЕ
  возвращается** (Кенан 12.05 уточнил: возвращается; противоречие — см.
  REFERRAL_RANKS_SPEC Q5).
- Покинуть страну во время активной войны — **нельзя**.

### Главком клана
- Топ-1 по `User.referralCount` (звание выше = главком).
- При равенстве — кто раньше вступил (`joinedAt ASC`).
- **Авто-смена** при 7 днях бездействия (требование Кенана 25.04; не реализовано).
- Главком имеет права:
  - **Approve/Reject** новых членов (open-join сейчас, менять в Фазе 3).
  - **Удалять** членов из клана.
  - **Объявлять войну** другому клану (нужен хотя бы 1 ≥ Лейтенант в клане).

### Объявление войны
- Длительность: `1 час / 1 день / 1 неделя / 1 месяц`.
- Лидер другого клана видит вызов: страна + размер кассы.
- При принятии — война на передний план списка.
- Бот публикует пост в публичный канал: «страна А vs страна Б, N часов/дней».
- Участники получают уведомление + полосу на HomePage.
- **Без офицера (никто ≥ Лейтенант) объявить нельзя.**

### Автоматические партии (matchmaking)
- Система раздаёт пары среди бойцов обеих стран.
- **Максимум 10 одновременных партий** (`MAX_CONCURRENT_BATTLES=10`).
- Завершилась одна — стартует следующая.
- Один игрок может побить «армию из 1000» — учитывается только сумма побед.
- Турнирные/военные сессии создаются `isPrivate=false` — попадают в публичное
  лобби с бейджем «⚔️ ВОЙНА».

### Финал войны
Побеждает страна с большим числом побед. **Распределение кассы проигравшей**:

| Доля | Кому |
|---|---|
| **20%** | 1-е место по победам в этой войне |
| **10%** | 2-е место |
| **5%** | 3-е место |
| **65%** | Остальные бойцы клана пропорционально вкладу (взнос+донаты) |
| **10% комиссия стола** | Платформе |

Каждому победителю уведомление: «Вы получили XXX,X K монет благодаря победе
вашей страны. 10% комиссия стола. На ваш счёт отправлено XXX,X K».

> **Текущее состояние кода (commit 0235a92):** распределение НЕ соответствует
> канону, сделано по весу `wins×1000+contribution`. Нужно переделать в Фазе 4.

### Командный турнир внутри клана
- Создаёт игрок (любой в клане).
- Длительность 1ч/1д/1н/1м.
- Касса = сумма взносов.
- Распределение как в клановых войнах.
- Без ограничения числа бойцов, но **10 одновременных партий**.

## 5. Режим Турниры

### Виды (автоматические по календарю)
- **Чемпион Недели** (`WEEKLY`)
- **Чемпион Месяца** (`MONTHLY`)
- **Чемпион Сезона** (`SEASONAL`) — квартал
- **Чемпион Года** (`YEARLY` или `WORLD`)
- **Чемпион Страны** (`COUNTRY`) — только для членов этой страны

### Вступление
- Любой игрок (кроме Чемпион Страны — нужно членство в стране).
- Взнос **10 000 ᚙ** одноразово. **Не возвращается** при выходе.
- Зрители могут донатить в кассу любую сумму.

### Поток
- Игрок выбирает любого участника турнира и отправляет вызов из турнира.
- Победы/поражения в турнире фиксируются в **профиле турнира** игрока (отдельно
  от глобального ELO).
- В Чемпионе Мира система **сама** распределяет вызовы (Swiss-bracket
  4/8/16/32 — `tournamentSwiss.ts`).
- Срок ответа на вызов — **24 часа**. Не ответил → автоматическое поражение.
- **3 пропуска подряд** → авто-вылет из турнира (не реализовано, нужно).

### Распределение призов (касса = взносы + донаты)

| Турнир | Победитель | Остаток |
|---|---|---|
| Чемпион Недели | **10%** | переходит в следующую неделю |
| Чемпион Месяца | **20%** | переходит в следующий месяц |
| Чемпион Сезона | **30%** | переходит в следующий сезон |
| Чемпион Года | **60% / 30% / 10%** (1, 2, 3 место) | **касса обнуляется** |

### Бейджи
Победителю — бейдж **«золотая лошадь в форме кубка»** (SVG) + дата + название
турнира. Вертикальный список в «Достижениях» профиля.

### Публикации в канал
- Еженедельно: имя чемпиона + статы + кнопка «вступить в следующую неделю».
- Раз в месяц / квартал / год — итоги с подробностями.

### Турнирные сессии в лобби
Турнирный матч создаётся как `Session(isPrivate=false, type=BATTLE)` с
`sourceType='TOURNAMENT'`. Видна в общем лобби с бейджем «🏆 ТУРНИР»
(SVG-кубок, без эмодзи). Участники видят кнопку «Принять», зрители —
«Смотреть».

## 6. Реферальная система — 18 рангов

### Принцип
- Игрок А приглашает игрока Б (любой реферальной ссылкой, включая ссылку на
  батл) → Б навсегда в команде А.
- Размер команды (`User.referralCount`) → **воинское звание** А.
- Звание определяет:
  - **разовый бонус** при активации каждого нового реферала;
  - **% lifetime-комиссии** с выигрышей членов команды (L1);
  - **право объявлять клановые войны** (от Лейтенанта);
  - **главком клана** = топ по званию в стране.

## 7. Магазин и финансы

### Внутренняя экономика (ᚙ)
- Welcome bonus новому игроку — **5 000 ᚙ всегда** (убрать Phase 1 gate в
  `economy.ts`).
- Покупка попыток: 1 000 ᚙ/шт.
- Покупка тем оформления: **10K – 1M ᚙ** (чем сложнее, тем дороже).
- Покупка аватаров, рамок, стилей доски (за монеты).
- После покупки темы — **мгновенное применение** (`setActiveTheme`).
- Внутренние операции **без комиссии**.

### TON / USDT
- Купить/продать ᚙ за TON или USDT.
- Подключение TON-кошелька (TonConnect) — разово **1 TON**.
- Вывод ᚙ на внешний TON-кошелёк:
  - комиссия системы **0.5%**
  - + комиссия блокчейна TON (с баланса игрока)
- Купля/продажа на P2P-бирже — **0.5% системе** с каждой операции.

### Темы оформления
Сейчас в коде 6 тем: `default, binance, chess_classic, neon_cyber, royal_gold,
matrix_dark`. Расширять до 10+ по росту цены 10K → 1M ᚙ.

## 8. Попытки (attempts)

- **3 попытки в день**, автоматическое восстановление +1 каждые 8 часов
  (00:00 / 08:00 / 16:00 UTC).
- Тратится при старте партии (BATTLE, BOT). FRIENDLY — без попыток.
- Можно докупить (1 000 ᚙ/шт) до 3 в день.
- В схеме есть `User.attemptSlots DateTime[]` для per-player таймера —
  пока не используется (Фаза 9).

## 9. Уведомления и каналы

### Toast (in-app)
- Длительность **3 секунды**.
- Тип: `info / success / error`.
- При первом входе на каждую страницу — **InfoPopup** (1 раз навсегда через
  Telegram CloudStorage + localStorage).

### Telegram-бот пушит
- победа партии,
- объявление войны,
- старт войны,
- призыв в турнир,
- активация реферала,
- топ-батл часа,
- итоги турниров.

### Публичный канал
- Час: топ-батл с max ставкой.
- Неделя/месяц/квартал/год: итоги соответствующих турниров.
- Старт каждой клановой войны.
